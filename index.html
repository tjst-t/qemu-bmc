Features

Everything a BMC should do

Full-featured virtual BMC packed into a single binary. No shell scripts, no supervisord, no extra daemons.

Redfish API

Full DMTF Redfish implementation — Systems, Managers, VirtualMedia, Chassis. Compatible with gofish client library.

HTTPS :443

IPMI over LAN

RMCP and RMCP+ with RAKP HMAC-SHA1 authentication and AES-CBC-128 confidentiality. Works with ipmitool and FreeIPMI.

UDP :623

VM IPMI (In-Band)

Guest OS IPMI via QEMU ipmi-bmc-extern KCS interface. MaaS commissioning scripts configure BMC users from inside the VM.

KCS :9002

noVNC Console

Browser-based VNC console served on the same HTTPS port. No extra VNC viewer software or additional ports needed.

/novnc/

QMP Control

Direct QEMU Machine Protocol integration for power operations, boot device changes, and VirtualMedia management.

UNIX socket

Single Binary

One Go binary replaces shell scripts + ipmi_sim + supervisord. Docker image includes QEMU with process management built in.

Quick Start

Up and running in 30 seconds

Docker Compose manages both QEMU and qemu-bmc in a single container.

terminal
# Create VM disk
$ mkdir -p vm
$ qemu-img create -f qcow2 vm/disk.qcow2 20G

# Start the container
$ docker compose up -d

# Check BMC status via IPMI
$ ipmitool -I lanplus -H localhost -U admin -P password mc info

# Check power state via Redfish
$ curl -k -u admin:password https://localhost/redfish/v1/Systems/1

# Power on the VM
$ ipmitool -I lanplus -H localhost -U admin -P password chassis power on
